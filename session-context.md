# Going UI — 세션 간 컨텍스트 유지

> **이 문서는 Claude Code 환경에서만 적용된다.** 채팅 환경에서는 파일 시스템 접근이 불가능하므로 해당 없음.

Going Library 프로젝트는 여러 세션에 걸쳐 작업하는 경우가 많다. 새 세션에서 이전 작업을 이어받으려면 **프로젝트 상태 파일**을 유지해야 한다.

---

## 프로젝트 상태 파일 (`project_status.md`)

프로젝트 루트에 `project_status.md` 파일을 생성하여 아래 내용을 유지한다.
**세션 시작 시 이 파일이 존재하면 반드시 읽고 현재 상태를 파악한 후 작업을 시작할 것.**

```markdown
## 프로젝트 개요
- 프로젝트명, 해상도, 통신 방식 등 핵심 정보 요약

## 완료된 작업
- [x] 완료된 항목 (날짜)

## 진행 중인 작업
- [ ] 현재 작업 중인 항목

## 이슈 사항
- 알려진 버그, 보류 사항, 사용자 확인 대기 항목

## 작업 노트
### 2026-03-19
- 작업 내용 기록
```

## 유지 규칙

1. **작업 시작 시** — `project_status.md`가 있으면 `Read`로 읽고 현재 상태 파악
2. **작업 완료 시** — 완료된 항목을 체크하고, 진행 상황을 업데이트
3. **이슈 발생 시** — 이슈 사항에 기록 (사용자 확인 대기, 알려진 버그 등)
4. **세션 종료 시** — 작업 노트에 해당 날짜의 작업 내용 기록

> 이 파일은 Claude Code가 자동으로 유지한다. 사용자가 별도로 관리할 필요 없음.
